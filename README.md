## 语落SayDone 是什么

**语落SayDone** 是一款 AI 驱动的语音/文字输入本地待办应用。核心理念：你只需要对着手机说一句话，AI 自动帮你解析出任务内容、日期、重要性、归属哪个池，并直接创建任务——无需手动填表。

**关键特点：**

- **语音优先**：长按说话，松手即创建，一条语音可同时创建、修改、删除多个任务
- **AI 自然语言理解**：说"明天买纸巾"会自动识别标题+目标日期，"顺手做"会自动归入轻任务池
- **纯本地存储**：所有数据存在手机 SQLite 中，无需注册账号，不上传云端
- **三池体系**：「日常待办」「轻任务」「长期目标」，各司其职
- **支持鸿蒙 4.2**：以 Android APK 形式安装运行

## 快速开始使用

### 第一步：获取 API Key

语落SayDone 依赖两个 AI 服务，需要你自行申请 API Key（应用不内置 Key）：

| 服务 | 用途 | 获取地址 |
|------|------|----------|
| **DeepSeek** | AI 自然语言解析（理解你说的"明天买纸巾"） | [DeepSeek 开放平台](https://platform.deepseek.com/api_keys) |
| **阿里云百炼（DashScope）** | 语音识别（将你说的话转成文字） | [阿里云百炼控制台](https://bailian.console.aliyun.com/cn-beijing?tab=model#/api-key) |

> DeepSeek 注册后即可获取 Key（`sk-...` 开头）。阿里云百炼需登录阿里云账号，在「模型服务 → API Key」页面创建 Key。

### 第二步：填入应用

1. 打开语落SayDone，点击底部 **「我」** → **「设置」**
2. 在「API Key」一栏填入 DeepSeek 的 Key，点击 **「测试连接」** 确认有效
3. 可选：在「模型」中切换 `deepseek-v4-flash`（推荐，速度快又便宜）或 `deepseek-v4-pro`（更强推理）
4. 在「语音识别（DashScope）」一栏填入阿里云百炼的 Key，点击 **「测试连接」** 验证

### 第三步：开始使用

- **语音输入**：在首页右下角**长按麦克风按钮**，说出你的任务（如"明天下午三点开会"），松手后 AI 自动解析并创建任务
- **文字输入**：**点击**麦克风按钮，在输入框中打字，点击发送即可

> 注：语音输入需要 DeepSeek Key + DashScope Key 都配置好。文字输入仅需 DeepSeek Key。

## 与普通 Todo 软件的区别

| 维度 | 普通 Todo 软件 | 语落SayDone |
|------|---------------|-------------|
| **输入方式** | 手动打字 → 逐字段填写（标题、日期、优先级、分类、备注...） | 说一句话 / 打一句话 → AI 自动解析并填完所有字段 |
| **数据归属** | 通常需要注册账号、云端同步 | 纯本地 SQLite 存储，无需注册，无云端 |
| **AI 能力** | 无 | 自然语言理解，一句话能创建/修改/删除/移动多个任务 |
| **任务组织** | 传统 Folder / List / Tag 模式 | 三池体系（日常排程 + 碎片顺手 + 长期规划），日期驱动的首页自动展示 |
| **创建效率** | 一条任务约 30 秒~2 分钟 | 一句话 5 秒，一条语音可同时建多条 |
| **轻量级随手记** | 需要切换到 App → 打字 → 选分类 → 保存 | 长按说话 → 松手完成，全过程不到 10 秒 |
| **提醒方式** | App 内弹窗或系统通知 | 系统通知栏本地推送，显示未完成数量 + 任务标题 |

核心区别概括：

- **传统 Todo** 是你**操作软件**：点→选→填→保存
- **语落SayDone** 是你**告诉软件**你想干什么：它帮你解析、分类、排程

## 解决了什么问题

1. **"打开就忘了要干嘛"**
   打开传统 Todo，面对空白输入框、各种分类、日期选择器……打字半天，早就忘了刚才想记录什么。语落SayDone 只需长按说话，2 秒搞定。

2. **"懒得打字，尤其在路上"**
   赶地铁、走路、开车等红灯——不可能停下来打字。长按说一句话即可。

3. **"一次有好几件事，不想一条一条建"**
   "今天写作业、做开发、写预习报告" → AI 自动拆成 3 条独立任务。

4. **"不知道该归哪个类、设什么优先级"**
   "顺手买纸巾" → AI 自动归入轻任务。"周五前写完很重要" → AI 自动标重要 + 设 DDL。你只管说，分类排程交给 AI。

5. **"任务跨天了不知道去哪了"**
   设定目标日期的任务，到期自动出现在首页。过期的进入"昨日未完成"，可一键移至今天。不用担心忘了某个任务。

6. **"不想注册账号、不想数据上云"**
   全部数据存在手机本地，换手机可通过 JSON 导出/导入迁移。零隐私顾虑。

7. **"普通 todo 首页是个大杂烩"**
   语落SayDone 首页按日期+池分层展示：今日必须做的、轻任务随机提醒、长期目标偶然看一眼、明日待办折叠可展开。信息密度高但不杂乱。

## 使用建议与技巧

### 如何说话让 AI 准确解析

**推荐的说法：**
> "明天下午三点开会"、"周五前写完报告，很重要"、"顺手买纸巾"、"长期目标学会 STM32 单片机"、"今天写作业、做 PPT、预习报告"

AI 不依赖特定关键词，而是**理解你的自然语义**。但你说话越完整，解析越准。

**区分 `target_date` 和 `DDL` 的技巧：**
- 你想表达"哪天去做" → 这个日期是 `target_date`
  - 例："明天买纸巾"（买是你要做的事） → `target_date = 明天`
- 你想表达"哪天有个事"或"哪天之前要搞定" → 这个日期是 `DDL`
  - 例："5月23号考C++"（考试是外部事件） → `DDL = 5月23号`
  - 例："周五前写完"（截止日期） → `DDL = 周五`

如果不确定怎么说，就按你平时跟人说话的方式说，AI 大多数情况能正确理解。

### 善用轻任务池

凡是"有空再说"、"不重要但可以做"、"碎片时间消遣"的事都放轻任务池：
- 买纸巾、查个价格、看看某篇公众号、回复某人消息……
- 在首页随机展示 5 条，点刷新换一批，起到"反正现在有空，随手做一条"的效果
- 轻任务池不分重要性，也不用设 DDL（移到轻任务池时自动清除）

### 合理利用日分界时间

如果你经常熬夜，把日分界时间调到凌晨 3:00 或 4:00。这样凌晨 2 点看到的是"今天"的任务，不会突然全部变过期。

### 定期清理回收站

默认 5 天自动清除，你也可以在设置里调整到 1-30 天。养成偶尔逛一下回收站、确认没误删的习惯。

### 换手机时迁移数据

1. 旧手机：**我 → 设置 → 数据导出**，通过微信/文件传输发送 JSON 文件
2. 新手机：安装语落SayDone → **我 → 设置 → 数据导入**，选择文件
3. 一般选「替换」，因为新手机是空的；如果新手机上已有数据再考虑「合并」

### 善用"明天做这些"

在日常待办池页面，进入选择模式，勾选几条任务，点「明天做这些」，这些任务的 `target_date` 统一设为明天。适合"今天太晚了，这些推到明天"的场景。

### 模型选择建议

- **日常使用**：`deepseek-v4-flash` 够用，速度快，成本极低（¥0.002/次左右）。所有模型均支持多轮上下文（修改/删除/移动任务）。
- **追求准确度**：偶尔切一次 `deepseek-v4-pro`，解析更稳定，适合复杂指令- **思考模式**：一般不需要开。如果你发现 AI 解析不准（比如多次把 DDL 和目标日期搞混），可以开思考模式提升准确度

## 功能详解与使用方式

> 以下为完整功能手册。如果你是第一次用，建议跳过本节先上手，遇到不明白的功能再回来查阅。

### 输入方式

#### 麦克风按钮

麦克风按钮位于首页右下角，圆形蓝色，所有主页面都可见。

| 操作 | 行为 |
|------|------|
| **点击** | 弹出文字输入框，上方显示三个快捷标签：「日常待办」「轻任务」「长期目标」。点击标签可选归属池，再输入文字发送 |
| **长按** | 进入语音录音模式。按钮放大 1.25 倍，边缘出现波纹动画，表示正在收音 |

#### 语音模式 — 四方向操作区

长按麦克风后，以按钮为中心弹出四个方向的操作区域：

| 方向 | 行为 | 图标 |
|------|------|------|
| **上方** | 日常待办 | 日历图标 |
| **左侧** | 轻任务 | 魔杖图标 |
| **左上角** | 长期目标 | 旗帜图标 |
| **下方** | 取消录音 | ✕ 图标 |

手指滑到对应区域松手 → 此任务归入该池。不滑走直接松手 → 不指定池，AI 自动推断或默认归入日常待办。

**录音反馈：**

- 录音时：按钮放大 + 波纹扩散动画
- 松手后：屏幕中下部出现半透明深灰**处理浮框**（tost），显示「处理中...」
- 成功 → 显示「已创建」/「已修改」→ 停留 2 秒 → 淡出消失
- 未识别到内容 → 显示「未识别到内容」→ 2 秒后淡出
- 断网/超时 → 显示「失败」→ 2 秒后淡出

#### 离线检测

按下麦克风按钮的瞬间即检查网络。如果当前没有网络连接，会提示「当前未联网」，不进入录音/文字输入界面，不发起任何网络请求。纯本地操作（编辑、完成、删除、迁移）不受离线影响。

#### LLM 处理队列

如果连续多次说话或输入文字，所有请求自动排队，LLM **串行处理**——一条处理完再处理下一条。每条都会显示自己独立的处理浮框，多条浮框堆叠显示，旧的在上，新的在下。

### 任务池体系

语落SayDone 用 **3 个任务池** 组织所有任务，每个任务只属于其中一个池：

| 池名称 | 用途定位 | 适合放什么 |
|--------|----------|------------|
| **日常待办** | 日常事务 + 排程 | 有明确时间的事：今天开会、明天交报告、周五前写完方案。可设目标日期，到期自动出现在首页 |
| **轻任务** | 不重要、有空随手做 | 碎片时间消遣：买纸巾、查个价格、看看某个网页。**不分重要性，不设 DDL** |
| **长期目标** | 长远规划 | 不适合放入日常的大目标：学会 STM32、健身三个月、读完某本书 |

#### 新任务默认归属

- 新创建的任务默认归入**日常待办**池
- 除非你通过快捷标签（文字模式）或手指滑向对应方向（语音模式）明确指定其他池
- AI 也可以根据你的措辞自动推断归属（如"顺手做"→ 轻任务，"长期目标"→ 长期目标）
- 在某个池的专属页面（从第二页「池」进入）中创建任务时，默认归入该池

#### 带日期任务的首页展示逻辑

日常待办池中的任务可以附带**目标日期**（`target_date`）。到那一天时，任务会**同步展示到首页「今日待办」板块**（任务本身仍然保留在日常待办池中）。

#### 过期未完成任务的处理

目标日期已过的任务，在首页会自动分区显示：
- 上半部分：**今日待办**（目标日期 = 今天的任务）
- 下半部分：**昨日未完成**（目标日期 < 今天的任务，标注分割线）

你可以进入选择模式，勾选过期任务后：
- **一键移至今天**：把目标日期更新为今天，任务继续在首页展示
- **移回日常待办**：清空目标日期，任务回归日常待办池，不再在首页展示

#### 日分界时间

默认**凌晨 4:00** 为新一天的开始（可在设置调整）。比如现在是 5 月 14 日凌晨 3:00，应用认为"今天"还是 5 月 13 日；凌晨 5:00 则"今天"是 5 月 14 日。这个设定影响「今日待办」「明日待办」「过期未完成」的判断。

### 任务属性

每个任务包含以下字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|:---:|------|
| 标题 | 文本 | ✅ | 任务名称，AI 从输入中提取 |
| 备注 | 文本 | ❌ | 补充说明，如"要用维达的"、"用 PDF 格式" |
| 创建时间 | 时间戳 | ✅ | 自动生成 |
| 目标日期 `target_date` | 日期 | ❌ | 计划做这件事的日期。到期当天出现在首页今日待办。AI 可解析"明天买纸巾" |
| 截止日期 `DDL` | 日期 | ❌ | 最后期限或事件日期。**仅影响排序优先级**，不会让任务出现在首页。有 DDL 自动标「重要」。AI 可解析"周五前写完" |
| 重要性 | 枚举 | ✅ | `普通` / `重要`。有 DDL 的任务自动标重要。重要任务显示 ⭐ 星标 |
| 归属池 | 枚举 | ✅ | `日常待办` / `轻任务` / `长期目标` |

#### `target_date` 与 `DDL` 的区别

这是 AI 解析准确与否的关键区分：

| 场景 | 填什么 | 例子 |
|------|--------|------|
| 你**打算某天亲身去做** | `target_date` | "明天买纸巾"、"今天做家务"、"后天取快递" |
| 某个**事件在某天发生**，或**某天之前必须完成** | `DDL` | "5月23号考C++"、"周五前写完报告"、"下周三之前背完单词" |

简单判断：你说的是"那天我去做"→ `target_date`；你说的是"那天有个事"或"那之前要搞定"→ `DDL`。

#### 排序规则

- **重要任务**：DDL 越近越靠前 → 同 DDL 按创建时间
- **普通任务**：按创建时间
- **轻任务池**：不区分重要性，统一按创建时间
- 首页「今日待办」板块：重要在前、普通在后

### 页面结构

应用共 **3 个主页面**，通过底部 Tab Bar 切换：

#### Tab 1 — 首页

```
┌─────────────────────┐
│  今日待办              │  ← 当前日期有 target_date 的任务
│  ┌───────────────┐   │     + 过期任务在「昨日未完成」分区
│  │ 任务卡片列表    │   │
│  └───────────────┘   │
├─────────────────────┤
│  轻任务                │  ← 从轻任务池随机抽取 5 条展示
│  ┌───────────────┐   │     [刷新] 按钮可重新随机
│  │ 可勾选完成      │   │
│  └───────────────┘   │
├─────────────────────┤
│  长期目标              │  ← 从长期目标池随机抽取 1-3 条
│  ┌───────────────┐   │     纯文本展示，不可直接完成
│  │ 仅标题文字      │   │     (点击不响应)
│  └───────────────┘   │
├─────────────────────┤
│  ▶ 明日待办（折叠）     │  ← 点击展开查看明天要做的任务
└─────────────────────┘
```

- 首页**仅用于浏览和完成勾选**，点击任务条目不响应（不会展开详情）
- 过期任务出现时，导航栏右侧出现「选择」按钮
- 内容超出屏幕 → 自动开放滑动；未满 → 锁定不可滑动
- 右下角悬浮麦克风按钮

#### Tab 2 — 池浏览页

三张卡片入口，点击进入对应池的专属页面：

- **日常待办** → 展示该池全部任务。顶部显示今日任务 + 分割线，下方按重要性排序。支持「明天做这些」按钮（选中任务统一设 target_date 为明天）
- **轻任务** → 展示该池全部任务，按创建时间排序
- **长期目标** → 展示该池全部任务

在池专属页面中，点击任务条目可**展开下拉详情区域**，显示完整备注、日期等字段，支持手动编辑。导航栏有「+」按钮可手动创建任务（默认归入当前池）。

#### Tab 3 — 我

```
我
├── 设置
│   ├── API Key              （DeepSeek，可测试连接）
│   ├── 模型                  （deepseek-v4-flash / v4-pro / kimi-k2.6）
│   ├── 思考模式               （开关）
│   ├── 语音识别 API Key       （DashScope，可测试连接 + 内置测试音频）
│   ├── 日分界时间             （默认 04:00）
│   ├── 未完成任务提醒         （默认 23:00，可关闭）
│   ├── 回收站保留天数         （默认 5 天）
│   ├── 数据导出               （JSON 文件，可分享/备份）
│   ├── 数据导入               （合并或替换现有数据）
│   ├── 开发者模式             （默认关闭）
│   └── 关于
├── 已完成任务              → 按原池分类展示，可点击撤销完成
└── 回收站                 → 按原池分类，显示剩余天数，可恢复/彻底删除
```

### 任务操作

#### 完成（勾掉）

每个任务条目左侧有一个**空心圆**。点击它以标记完成：

1. **点击空心圆** → 圆圈变为实心（带白色 ✓）+ 字体变灰 → 任务下移至底部「已完成：N 条」分割线下方
2. 每新增一个标记，**1 秒倒计时重置**。持续 1 秒无新标记 → 所有已完成条目 + 分割线整体淡出 0.6 秒 → **归档到「已完成任务」**
3. 淡出期间如果再次点击标记 → **取消淡出**，恢复显示，重新计时

#### 撤销完成

归档前在首页直接撤销：
- 在分割线下方，**点击已实心的圆圈** → 圆圈恢复空心 + 字体恢复 → 任务回归原位置

归档后在「已完成任务」页撤销：
- 所有已完成任务圆圈默认为**实心**
- **单选撤销**：点击实心圆 → 圆圈变空心 → 条目淡化消失 → 任务回归原池
- **选择模式**：可滑动多选，批量撤销完成

#### 选择模式

进入选择模式后，圆圈变为方框（勾选框），底部出现**「灵动岛」**选择栏（悬浮圆角卡片，与屏幕边缘有间距，按钮水平均分）。

支持的操作：

| 操作 | 说明 |
|------|------|
| **全选 / 全不选** | 一键选中当前列表全部条目 |
| **删除** | 选中任务进入回收站 |
| **移动到其他池** | 弹出池选择，迁移选中任务 |
| **明天做这些** | （日常待办池页面）统一将选中任务的 target_date 设为明天 |
| **一键移至今天** | （首页过期任务）更新 target_date 为今天 |
| **移回日常待办** | （首页过期任务）清空 target_date，回归日常待办池 |
| **批量撤销** | （已完成任务页）批量恢复选中任务 |

#### 滑动多选 / 滑动完成

在条目左侧圆圈位置按住并**纵向滑动**：

- **首页（非选择模式）**：手指滑过的条目即时标记完成（变实心✓）/ 回滑自动取消。松手后统一进入完成流程。采用**橡皮筋逻辑**：固定端 + 拖拽端覆盖范围内的条目自动标记，**统一状态翻转**（未完成→已完成，已完成→未完成）
- **选择模式**：手指滑过的条目自动加入/移出选中集合
- **已完成任务页（选择模式）**：支持滑动多选批量撤销

#### 删除与回收站

- 被删除的任务进入**回收站**，按**原池分类**展示
- 回收站按**丢弃时间降序**排列（最近删除的在最前）
- 每条任务显示**剩余保留天数**（默认 5 天，可在设置调整）
- 支持**恢复**（回到原池）或**立即彻底删除**
- 回收站支持**选择模式**，可多选后一键彻底删除
- 每次打开 App 和打开回收站页面时，自动清理已过期的任务

#### 已完成任务页面

- 按**原池分类**展示
- 支持按**完成时间排序**（正序/倒序，点击导航栏左侧排序图标切换）
- 所有任务圆圈默认为**实心**
- 点击实心圆 → 撤销完成（带回原池）
- 支持选择模式下的批量撤销和删除

### AI 能力边界

#### AI 能做什么

| 能力 | 说明 | 示例 |
|------|------|------|
| **创建任务** | 从一句话中提取标题、日期、重要性、归属池、备注 | "明天买纸巾" → 标题="买纸巾"，target_date=明天，pool=daily |
| **多任务拆分** | 一句话包含多个任务时，自动拆分为多条 | "今天写作业、做开发、写预习报告" → 3 条独立任务 |
| **修改已有任务** | 通过多轮对话自动获取任务上下文，匹配指代的任务并修改字段 | "把买纸巾的 DDL 改成五月三十号" → AI 获取任务列表 → 找到"买纸巾" → 更新 DDL |
| **删除任务** | 指代已有任务并要求删除 | "删掉买纸巾那个" |
| **移动任务** | 将已有任务移到其他池 | "把写报告移到轻任务里" |
| **混合操作** | 一条语音可同时包含创建+修改+删除+移动 | "嗯那个，买纸巾删了吧，C++ 作业明天做，写报告改成重要" |

**修改/删除任务的说话示例：**

| 你这样说 | AI 会做的事情 |
|---------|------------|
| "把买纸巾的 DDL 改成五月三十号" | 获取上下文 → 找到"买纸巾" → 修改 DDL |
| "那个写报告的帮我移到轻任务去" | 获取上下文 → 找到"写报告" → 移动池 |
| "删掉取快递那条" | 获取上下文 → 找到"取快递" → 删除 |
| "看书那个，目标日期删了，ddl 设五月三十，然后挪轻任务" | 获取上下文 → 一次 modify 全改（清日期+设ddl+移池） |
| "C++ 作业明天做，然后把买纸巾那条备注加上'要用维达'" | 获取上下文 → 修改两个不同任务各自的字段 |

> 不需要记命令格式，像平时说话就行。AI 会自动获取任务列表，匹配你说的任务名，执行修改。

#### AI 不能做什么

- 不能闲聊：问候、天气等非任务输入会返回"未识别到任务内容"
- 不能写作文、翻译、总结等通用 AI 问答
- 不能在没有明确指代的情况下修改任务：如果说不清"哪个任务"，AI 会要求上下文

#### 多轮对话机制（v1.0.2 起）

当你说"把买纸巾的 DDL 改成五月三十号"时，AI 发现自己没有任务列表上下文：
1. AI 调用 `get_task_context()` 获取所有活跃任务
2. 系统将任务列表注入对话
3. AI 遍历匹配"买纸巾" → 调用 `modify` 修改对应字段
4. 最多 3 轮对话，防止死循环
5. 纯新建任务（如"明天买纸巾"）仍为单轮，无额外开销

### 开发者模式

在「我 → 设置 → 开发者模式」开启后，可在「自定义内容」中修改以下内容：

| 自定义项 | 说明 |
|----------|------|
| **池名称** | 把"日常待办"改为"工作待办"，"轻任务"改为"碎片"，"长期目标"改为"目标"等 |
| **空状态文案** | 各池为空时的占位提示文字（5 个文案各自定义） |
| **按钮文字** | 「明天做这些」按钮的显示文字 |
| **Toast 文字** | 「处理中...」「已创建」「已修改」等浮框提示文字 |

> 修改仅影响界面显示和 AI 行为，不影响底层数据标识（`daily`/`light`/`longterm` 永远不变）。

**log 浮窗**：开启开发者模式后，屏幕上出现一个可拖动的小窗口，显示：
- 队列状态（队列中待处理数）
- 当前处理中的文字
- 最近 15 条操作日志（10 秒自动过期）
- 可最小化为小图标
